# Design Principles

These principles define how this system should behave over time.

---

## 1. Human-first input

The user should NOT need to organize before writing.

Friction must remain low.

---

## 2. AI handles structure

- indexing
- summarization
- relation mapping
- prioritization

---

## 3. Avoid over-classification

Folders are:
- simple
- shallow
- stable

Precision lives in:
- summaries
- metadata
- relations

---

## 4. Layered retrieval

Never rely on full-vault reading.

Preferred order:
1. index
2. summary
3. full text

---

## 5. Knowledge is multi-dimensional

Do NOT force notes into a single category.

Use:
- tags
- relations
- context layers

---

## 6. Everything must be rebuildable

Indexes and summaries are disposable.

Original notes are the source of truth.

---

## 7. This is a system, not a folder

This vault is:
→ a knowledge operating system

Not:
→ a static file structure
